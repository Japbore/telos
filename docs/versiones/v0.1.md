### Casos de Uso (Paso a Paso)

#### CU-01: Alta de Libro y Test
1. El usuario entra en la app y no hay libro activo.
2. Ingresa título ("El Quijote") y páginas ("500").
3. Hace el test de velocidad del lector: pulsa "Iniciar cronómetro" y al rato "Parar", registrando p.e. `01:30` (90 seg / pag). Guarda el estado.
4. El libro queda `activo` y se muestra el panel de progreso principal.

#### CU-02: Registrar avance diario
1. El usuario entra a la pantalla del libro activo.
2. Añade un nuevo avance eligiendo Fecha = Hoy y Útima Página = 45.
3. Se guarda en la BD. La app recalcula e imprime los resultados.
4. Si el mismo día vuelve a leer y alcanza la 60, guarda nuevamente Fecha = Hoy y Última Página = 60. El registro anterior se actualiza a 60.

#### CU-03: Visualización de Estimaciones
1. **Tiempo dedicado:** Si la última página es 60, y tarda 90 seg por pag, el tiempo total dedicado de lectura que se asume es de `60 * 90 = 5400 seg` = 1.5 horas.
2. **Tiempo restante:** Quedan `440 * 90` seg.
3. **Fecha estimada de fin:** Para esto, la app mide el ritmo promedio basándose en la evolución de las fechas (pags añadidas por día activo). Ej: Si pasaron 3 días de lectura y se leyeron 60 pgs, el promedio es 20 pags/día activo. Quedan 440 págs / 20 = 22 días de lectura restantes. Sumados al último día logueado, proyecta un fin de libro en el calendario.

#### CU-04: Terminar o Eliminar (Reset)
1. Si el usuario guarda un log donde la última página es igual a `totalPages`, se le muestra enhorabuena y el botón para terminar/archivar el libro, liberando la app para uno nuevo.
2. En cualquier momento el usuario pulsa "Resetear libro", se hace un *hard delete* o cambio de estado a *reset* de ese libro y se devuelve al CU-01.
